# 文献综述翻译计划（面向 Loop Tree + 源码归因课题）

## 1. 课题定位（用于文献综述首页）

建议把课题定位为：

- 面向 Ascend NPU 分布式推理的时序循环建模与源码归因性能分析

文献综述的目标不是泛泛介绍“NPU性能分析”，而是回答三个具体问题：

1. 如何把原始 trace 压缩为可读、可统计的结构化表示（loop/tree/macro）。
2. 如何从结构化表示中做节点级性能归因（wait/comm/exec/idle、跨 stream 关系）。
3. 如何把性能模式映射回源码位置，形成可执行优化建议。

## 2. 翻译型综述的交付目标

每篇核心文献的翻译/整理都要产出统一四元组：

1. 问题定义：它解决什么 profiling/trace 分析问题。
2. 方法要点：核心建模方式（图、序列、约束优化、统计模型等）。
3. 可迁移点：对本课题可直接借鉴的技术。
4. 局限与空白：本课题将如何补足。

最终综述要能支撑你后续“方法章节”的三项设计：

- Loop Tree 表达
- 节点聚合统计
- 源码归因链路

## 3. 建议阅读主题与优先级

### P0（必须优先）

1. 运行时轨迹与可视化基础
- Perfetto / Chrome Trace 事件模型与时序关系
- GPU/NPU stream、event、host-device 同步语义

2. LLM 推理执行机制
- vLLM 调度（continuous batching）、KV/cache、prefill/decode 阶段
- 分布式推理中的通信-计算关系（TP/PP）

3. 模式挖掘与压缩表达
- 序列重复模式挖掘、语法压缩（grammar-based）思想
- 层次化模式（macro/子树复用）的表示方法

### P1（强相关）

1. 异构系统性能归因
- critical path / queueing / overlap 分析
- host API 调用到 device task 的关联方法

2. 源码级性能映射
- 从算子名/kernel 名/调用栈到源码位置的映射策略
- 证据与置信度建模（而不是单点猜测）

### P2（补充）

1. 自动化性能分析报告生成
- 规则 + 模型联合解释
- 结论可验证性与可复现性

## 4. 建议阅读清单（起步版）

第一批建议 12~18 篇，结构上按“基础 -> 方法 -> 应用”组织：

1. Trace/Profiler 体系（3~4 篇）
- Perfetto 相关文档/论文
- PyTorch Profiler / Kineto 机制
- Nsight Systems（事件/流/同步语义）

2. 模式挖掘与压缩（4~5 篇）
- Grammar-based 序列压缩代表工作
- Frequent episode / motif 发现代表工作
- 层次模式表示与可解释化工作

3. 异构归因与关键路径（3~4 篇）
- CPU-GPU 协同关键路径分析
- stream overlap / queue delay 建模方法

4. LLM serving 与分布式推理（3~5 篇）
- vLLM/PagedAttention
- 分布式推理通信优化相关论文

## 5. 综述章节建议（可直接映射到 LaTeX）

建议综述正文结构：

1. 研究背景与问题定义
2. 相关技术脉络
- 2.1 Trace 表示与时间线分析
- 2.2 序列模式挖掘与层次压缩
- 2.3 异构系统性能归因
- 2.4 源码映射与可解释报告
3. 现有工作的不足
4. 本课题技术路线与创新点
5. 小结

## 6. LaTeX 模板落地建议（基于 document-translation-template）

当前模板核心文件：

- `Experimental_Report.tex`
- `Experimental_Report.bib`
- `Experimental_Report.cls`

建议你保留 cls，不改样式，只替换 tex 内容骨架：

1. 标题改为课题相关题目（文献综述/外文翻译题名）。
2. `abstract` 写成“综述目的 + 方法 + 主要结论”。
3. 主体按第 5 节的结构填充。
4. 参考文献统一走 `.bib`，避免手写列表。

## 7. 两周执行节奏（可压缩）

1. 第 1-3 天：完成 P0 阅读与笔记卡片（每篇 1 页）。
2. 第 4-7 天：完成 P1 阅读并写“对本课题可迁移点”。
3. 第 8-10 天：形成综述初稿（先中文）。
4. 第 11-14 天：补图表、统一术语、修订引用格式。

## 8. 写作质量红线

1. 不做“工具清单式综述”，必须围绕你的三个核心问题组织。
2. 每节都要有“对本课题的启发/限制”小结。
3. 任何“源码归因”表述都强调证据链与置信度，避免绝对化措辞。
